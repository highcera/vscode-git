다음은 데이터베이스에 대해서 알아보겠습니다
파이어베이스에서 사용할 수 있는 데이터베이스는 Firestore 그리고 Realtime Database 가 있습니다.
둘다 데이터베이스의 실시간업데이트를 사용할 수 있지만, 개념적으로 사용하는 방법이 많이 다릅니다.
우선, 피이어스토어의 경우엔 무한적으로 key-value 매칭이 불가능하고 collection과 document라는 특정 개념으로 문서가 묶여 데이터가 저장되는 특징을 가지고있습니다.

그러나 실시간데이터베이스(Realtime Database)는 key-value의 무한 매칭이 가능하고, 데이터를 쉽게 저장하고 삭제하고 수정할수있는 이점이 있습니다.

저희는 firestore보다 한층더 직관적이고 단순 key-value 매칭의 쉬운 Realtime Database 를 사용해보도록 하겠습니다.

먼저 좌측 메뉴에서 Realtime Database를 클릭해 들어가줍니다.
그리고 데이터베이스 생성하기를 클릭합니다.
실시간 데이터베이스의 위치를 먼저 물어보게되는데요, 아직까지 실시간데이터베이스는 벨기에와 미국밖에 지원하지 않으므로, 미국을 선택하고 다음을 눌러줍니다.

다음으로 데이터베이스를 생성하기전에 해당 데이터베이스를 현재 테스트용도로 사용할 것인지, 프로덕 상태로 사용할 것인지를 물어보게됩니다.
후에 '규칙' 이라는 부분을 수정해서 특정 데이터베이스 액세스권한을 조정할수있는 기능이 '규칙' 기능입니다.
현재는 테스트 용도로 사용할 예정이니, 테스트를 클릭하고 데이터베이스를 만들어줍시다
데이터베이스가 다 만들어졌으면 한번 들어가서 봅시다.

먼저 규칙 탭을 클릭해줍니다.
규칙탭을 보면 세팅된 코드들이 있습니다.
이 코드들은 해당 timestamp까지 오픈이 되어있다는걸 말합니다. 오른쪽에 보시면 주석으로 몇월 몇일까지 가능한지 나와있죠 해당 timestamp가 오른쪽 주석에 적힌 날짜를 의미합니다.
해당 규칙에 구애받지않기위해서 read와 write부분에 true를 줘서 모두 액세스 가능한 상태로 두겠습니다.
true로 변경하시고, 저장을 눌러줍니다.
아, JSON 타입이므로 콤마를 잊지마세요!
그리고난뒤 데이터로 돌아와서 임의의 데이터를 한번 만들어보겠습니다

먼저 더하기를 누른뒤, key값을 적고 value를 적어줍니다.
