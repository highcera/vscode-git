09 n8n 커뮤니티 노드
n8n 커뮤니티 노드는 개발자들이 만든 확장 기능으로, 기본 노드로는 구현할 수 없는 특수한 기능들을 제공합니다. 
현재 2,800개 이상의 커뮤니티 노드가 있으며, 지속적으로 증가하고 있습니다.

커뮤니티 노드 개요
주요 특징
확장성: 기본 n8n 기능을 크게 확장
다양성: AI, 데이터 처리, 통신, 자동화 등 다양한 분야
오픈소스: 대부분 MIT 라이선스로 무료 사용 가능
커뮤니티 기반: 개발자들의 실제 필요에 의해 제작

설치 방법
# 커뮤니티 노드 설치
npm install n8n-nodes-[노드명]

# 예시: Backup Credentials 노드 설치
npm install n8n-nodes-backup-credentials
n8n 인터페이스에서 설치 1. Settings → Community Nodes 2. 노드 패키지명 입력 3. Install 버튼 클릭

import { IExecuteFunctions } from 'n8n-core';
import {
  INodeExecutionData,
  INodeType,
  INodeTypeDescription,
} from 'n8n-workflow';

export class MyCustomNode implements INodeType {
  description: INodeTypeDescription = {
    displayName: 'My Custom Node',
    name: 'myCustomNode',
    group: ['transform'],
    version: 1,
    description: '사용자 정의 노드',
    defaults: {
      name: 'My Custom Node',
    },
    inputs: ['main'],
    outputs: ['main'],
    properties: [
      {
        displayName: 'API Key',
        name: 'apiKey',
        type: 'string',
        default: '',
        required: true,
      },
    ],
  };

  async execute(this: IExecuteFunctions): Promise<INodeExecutionData[][]> {
    const items = this.getInputData();
    const returnData: INodeExecutionData[] = [];

    for (let i = 0; i < items.length; i++) {
      const apiKey = this.getNodeParameter('apiKey', i) as string;

      // 커스텀 로직 구현
      const result = await this.processData(items[i].json, apiKey);

      returnData.push({
        json: result,
      });
    }

    return [returnData];
  }

  private async processData(data: any, apiKey: string) {
    // 실제 처리 로직
    return {
      processed: true,
      data: data,
      timestamp: new Date().toISOString()
    };
  }
}

// 입력 검증 함수
private validateInput(data: any): boolean {
  if (!data || typeof data !== 'object') {
    throw new Error('Invalid input data');
  }

  // 필수 필드 검증
  const requiredFields = ['id', 'name'];
  for (const field of requiredFields) {
    if (!data[field]) {
      throw new Error(`Missing required field: ${field}`);
    }
  }

  return true;
}

async processBatch(items: INodeExecutionData[]): Promise<any[]> {
  const batchSize = 100;
  const results = [];

  for (let i = 0; i < items.length; i += batchSize) {
    const batch = items.slice(i, i + batchSize);
    const batchResults = await this.apiCall(batch);
    results.push(...batchResults);
  }

  return results;
}
2. 캐싱 전략
// 간단한 메모리 캐싱
private cache = new Map<string, any>();

async getDataWithCache(key: string): Promise<any> {
  if (this.cache.has(key)) {
    return this.cache.get(key);
  }

  const data = await this.fetchData(key);
  this.cache.set(key, data);

  // 5분 후 캐시 만료
  setTimeout(() => this.cache.delete(key), 5 * 60 * 1000);

  return data;
}

왜 중요한가요? - 나중에 봤을 때 "이게 뭐 하는 노드지?" 안 헷갈려요 - 다른 사람이 봐도 이해하기 쉬워요

🔧 문제 해결 꿀팁
❗ 자주 하는 실수 TOP 5
1. "노드가 실행이 안 돼요!"
확인해보세요: - ✅ 워크플로우가 "Active" 상태인가요? (우측 상단 토글 켜기) - ✅ 모든 노드가 화살표로 연결되어 있나요? - ✅ 필수 입력값에 빨간 표시 없나요?

해결법: 1. 노드 하나씩 "Test Step" 버튼으로 테스트 2. 에러 메시지 읽어보기 (대부분 뭐가 문제인지 알려줘요) 3. 노드 설정에서 빨간색 테두리 찾기 → 거기 값 입력

2. "Credential 연결이 안 돼요!"
확인해보세요: - ✅ API 키나 비밀번호를 정확히 복사했나요? (공백 주의!) - ✅ 외부 서비스에서 API 접근을 허용했나요? - ✅ Credential 만료되지 않았나요?


해결법: 1. Settings → Credentials 가기 2. 해당 Credential 클릭 → Delete 3. 다시 새로 만들기 4. 복사-붙여넣기할 때 앞뒤 공백 확인

3. "데이터가 이상하게 나와요!"
확인해보세요: - ✅ 이전 노드의 출력 형태를 봤나요? (노드 클릭 → 우측에서 OUTPUT 탭 확인) - ✅ {{ $json.필드명 }} 에서 필드명이 정확한가요?

해결법: 1. 문제되는 노드 앞에 "Set" 노드 추가 2. Set 노드에서 데이터 구조 확인 3. 필요한 필드만 골라서 깔끔하게 정리

팁! 데이터가 배열인지 객체인지 헷갈리면: - 배열: [{}, {}, {}] 여러 개 - 객체: {} 하나

4. "워크플로우가 너무 느려요!"
원인과 해결: 1. API 호출이 많을 때 - 해결: 꼭 필요한 데이터만 가져오기 - 불필요한 필드는 제외

대용량 데이터 처리
해결: "Split In Batches" 노드로 나눠서 처리
한 번에 100개씩만 처리하도록

복잡한 코드

해결: Code 노드 대신 기본 노드 활용
예: IF 노드, Filter 노드가 더 빠름
5. "테스트는 되는데 실제론 안 돼요!"

확인해보세요: - ✅ Webhook URL이 HTTPS인가요? (HTTP는 안 됨) - ✅ 트리거 조건이 맞나요? (시간, 조건 등) - ✅ 실행 기록을 봤나요? (Executions 탭에서 확인)

해결법: 1. 우측 상단 "Executions" 클릭 2. 실패한 실행 찾기 3. 에러 메시지 확인 4. 해당 노드 더블클릭 → 설정 수정

💡 1. Snippets로 템플릿 즉시 적용
어떻게 유용한가요? - n8n에서 자주 쓰는 Expression을 매번 타이핑하지 않아도 돼요 - 복잡한 JSON 스키마를 단축키로 바로 입력 - 실수 없이 정확한 코드 삽입

설정 방법:


Raycast 열기 (Cmd + Space 또는 설정한 단축키)
"Create Snippet" 검색
자주 쓰는 코드 저장
n8n 필수 Snippets 추천:

Snippet 1: 현재 시간 Expression
Keyword: ;now
Content: {{$now.toFormat('yyyy-MM-dd HH:mm:ss')}}
언제 쓰나요? 로그 기록, 타임스탬프 추가할 때

Snippet 2: 조건부 값
Keyword: ;ifelse
Content: {{$json.field ? $json.field : 'default'}}
언제 쓰나요? 값이 없을 때 기본값 설정할 때

Snippet 3: JSON 파싱
Keyword: ;parse
Content: {{JSON.parse($json.rawData)}}
언제 쓰나요? 문자열을 JSON으로 변환할 때

Snippet 4: 배열 필터
Keyword: ;filter
Content: {{$json.items.filter(item => item.status === 'active')}}
언제 쓰나요? 배열에서 특정 조건 데이터만 추출할 때

Snippet 5: Code 노드 템플릿

Keyword: ;code
Content:
const items = $input.all();

return items.map(item => {
  return {
    json: {
      // 여기에 로직 작성
      result: item.json.data
    }
  };
});
언제 쓰나요? Code 노드에서 데이터 변환할 때

Snippet 6: 에러 핸들링
Keyword: ;try
Content:
try {
  // 실행할 코드
  const result = $json.data;
  return [{ json: { result } }];
} catch (error) {
  return [{ json: { error: error.message } }];
}
언제 쓰나요? Code 노드에서 안전하게 에러 처리할 때

팁! Keyword는 ;로 시작하면 실수로 실행될 일이 없어요!
